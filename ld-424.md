## 项目主要需求

### 项目概述

【纯后端服务】科研试剂与耗材库存管理 API 服务。面向实验室提供试剂和耗材的采购、入库、领用、盘点和预警的 RESTful API，支持危险化学品的特殊管理流程。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿后端全链路）

1. **试剂（Reagent）** — 字段：试剂名称、CAS号、分子式、分子量、纯度等级、危险等级（Safe/Irritant/Corrosive/Flammable/Toxic/Explosive）、存储条件（RoomTemp/Refrigerated/Frozen/Dark/Ventilated）、供应商ID、当前库存量、单位（g/mL/L/mol/bottle）、最低库存阈值、存放位置、MSDS链接
   - 关联：1种试剂 → 多条入库记录、多条领用记录
   - **贯穿后端**

2. **耗材（Consumable）** — 字段：耗材名称、型号、分类（Glassware/Plasticware/Filter/Pipette/Tube/Other）、当前库存量、单位、最低库存阈值、存放位置、单价
   - **贯穿后端**

3. **入库记录（StockInRecord）** — 字段：物品ID（试剂或耗材）、物品类型（Reagent/Consumable）、入库数量、批号、生产日期、过期日期、采购单号、入库人ID、入库日期、质检结果（Pass/Fail/Skip）
   - **贯穿后端**

4. **领用记录（UsageRecord）** — 字段：物品ID、物品类型、领用人ID、领用数量、领用日期、所属实验ID（可选）、用途说明、审批状态（Pending/Approved/Rejected）、审批人ID
   - **贯穿后端**

5. **盘点记录（InventoryCheck）** — 字段：盘点日期、盘点人ID、盘点状态（InProgress/Completed/Discrepancy）、盘点范围描述
   - 关联：1次盘点 → 多条盘点明细（InventoryCheckItem：物品ID、系统库存、实际库存、差异、差异原因）
   - **贯穿后端**

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / LabManager / Researcher / Student
   - 触达文件层：DB role表 + RBAC中间件 + controller权限检查 + 类型定义
   - 特殊规则：危险等级为Toxic/Explosive的试剂，Student不能自行领用，必须LabManager审批

2. **审计日志**
   - 记录入库、领用、盘点、危险品特殊操作
   - 触达文件层：DB audit_log表 + 日志中间件 + service埋点 + 类型定义

3. **接口限流**
   - 触达文件层：限流中间件 + Redis

### 共享枚举/常量（≥ 2 组）

1. **危险等级 HazardLevel**：Safe / Irritant / Corrosive / Flammable / Toxic / Explosive
   - 定义：`backend/src/types/enums.ts`

2. **领用审批状态 UsageStatus**：Pending / Approved / Rejected

3. **存储条件 StorageCondition**：RoomTemp / Refrigerated / Frozen / Dark / Ventilated

4. **物品类型 ItemType**：Reagent / Consumable

5. **质检结果 QCResult**：Pass / Fail / Skip

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色权限校验（含危险品特殊规则）
3. **`auditLogMiddleware`** — 审计日志
4. **`errorHandlerMiddleware`** — 统一异常
5. **`rateLimitMiddleware`** — 接口限流
6. **`requestLoggerMiddleware`** — 请求日志
7. **`validationMiddleware`** — 参数校验

### 技术要求

- **英文短名（用于 Docker Compose）**：`reagent-stock-api`
- **后端端口**：19303 / 3000
- **后端**：NestJS + TypeScript
- **数据库**：PostgreSQL 15
- **ORM**：TypeORM
- **缓存**：Redis 7（限流 + 低库存预警缓存）
- **认证**：JWT
- **API文档**：Swagger

### 部署要求

后端 + PostgreSQL + Redis，Docker Compose 编排。
- 顶层 `name: reagent-stock-api`
- 不写 `version:` 字段
- PostgreSQL + Redis 命名卷 + PostgreSQL healthcheck
- `.env` 必含：`COMPOSE_PROJECT_NAME`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `JWT_SECRET`, `REDIS_URL`, `BACKEND_PORT`

### Git 要求

`git init` + `.gitignore`。
